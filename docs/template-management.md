LandPPT 的模板管理中心是一个“HTML 模板仓库 + AI 生成 + 导入导出”集合页。本文按照「如何快速找到模板」「如何创建/生成模板」「如何保障质量」三个维度来说明。

## 模块概览

- **模板宫格**：展示缩略图、标签、使用次数、创建人、默认标记。hover 时显示操作按钮。
- **工具栏**：提供“新建模板”“AI 生成模板”“导入模板”“刷新”“跳转模板广场”等入口。
- **筛选/分页**：支持关键字搜索、标签筛选、分页与每页条数（6/12/24/48）切换。

## 常见操作

### 1. 查找与预览

- 输入关键字实时筛选模板名；标签下拉列出常用标签，便于分类检索。
- 每个卡片都带有 16:9 预览 iframe，会自动注入示例数据；失败时显示“暂无预览”占位。
- 点击“预览”可打开全屏 modal，使用和 PPT 编辑器一致的渲染方式，方便确认细节。

### 2. 新建/编辑模板

| 字段 | 说明 |
| ---- | ---- |
| 模板名称 (`template_name`)* | 必填，唯一标识 |
| 描述 (`description`) | 用于团队协作的文字说明 |
| 标签 (`tags`) | 逗号分隔，推荐按照场景/行业命名 |
| 设为默认 (`is_default`) | 全局仅能有一个默认模板，默认模板无法删除 |
| HTML 模板 (`html_template`)* | 建议包含标准 `<!DOCTYPE html>` 结构，并预留 `{{ title }}`、`{{ content }}`、`{{ slide_number }}` 等占位符 |

模板表单底部提供「预览」「取消」「保存」。保存后列表自动刷新。

### 3. AI 生成模板

1. 填写提示词（风格、颜色、布局等）。
2. 定义模板名称、描述、标签。
3. 选择生成模式：
   - 📝 纯文本：仅使用提示词。
   - 🎨 参考风格：上传图片作为灵感。
   - 🔄 1:1 还原：尽量复刻上传图片的布局。
4. 上传参考图（可选，支持 JPG/PNG/WebP ≤10MB）。
5. 提交后进入进度面板（30~60 秒）；可随时取消。
6. 生成完成后展示 iframe 预览与「AI 思考过程」（可复制原始响应），满意后保存。

### 4. 导入/导出

- **导入**：选择 `.html` 或 `.json` 文件。`json` 需包含 `template_name`、`description`、`html_template`、`tags` 等字段。
- **导出**：卡片上的「导出」按钮将当前模板序列化为 JSON，并附加 `export_info`（导出时间、原始 ID），可在其它实例中导入。默认不保留 `is_default`，避免覆盖现有默认模板。

### 5. 卡片操作

- **设为默认**：按钮置灰即说明当前模板为默认模板；系统全局仅允许一份默认模板。
- **复制**：输入新名称即可生成副本，适合做 A/B 变体。
- **删除**：需二次确认，默认模板不能删除。
- **导出 HTML**：在右键菜单或卡片按钮中，可导出单个模板的 HTML，用于外部审查或提交给设计同学。

## 面向设计/开发者的建议

- **标签规范**：建议使用「行业-风格」形式（例如 `finance-modern`），方便前端下拉快速定位。
- **HTML 规范**：固定 1280×720 画布、使用相对路径/内联资源、避免直接引用第三方脚本，可减少跨域和 CSP 风险。
- **AI 模板审查**：生成后的模板最好先在测试环境预览，确保 CSS/JS 没有额外依赖。
- **版本管理**：可将导出的 JSON 存入 Git 仓库，便于回滚；对于默认模板，建议保留多个备份。
- **与编辑器联动**：PPT 编辑器会读取默认模板。若模板包含新增的自定义变量，请在编辑器中同步渲染逻辑。

通过模板管理中心，你可以构建企业级模板库，结合 AI 生成、导入导出与默认模板机制，在不同项目中快速复用统一风格。
